问题：我在想手机打卡如何限制不让同事帮忙打卡
难点（推翻）：你需要在前台或者java端判断 ios 或者 安卓系统 然后获取各自的手机序列号
难点 ： 需要获取手机imie码 这个码是唯一的 https://blog.csdn.net/chaozhung_no_l/article/details/8930404
      但是没有测试ios与安卓 获取这个imie码是否这个方法公用
      不过程序就是这样有思绪有路走就能走通 怕的是走错路走的不对不敢走

方案一：（答案接近 但不健全）
把手机登录接口之前限制个查询
登录需要把本机序列号与账户绑定
登录时提示 手机只能绑定一个账号
这个是以账号为唯一键
登录的时候根据账户 查询数据库
if(null){
	插入账号 与 序列号
}else{
	if(序列号 == 序列号){
           成功登录
	}else{
	       你想打卡？门都没
	}
}
 败笔：可是如果手机丢失 换手机情况下
 只能新做接口让把账号连接删除 或者  线下删除数据库表数据

方案二：（不算完美...可以让不是公司的朋友帮忙打卡（不行就再绑个手机卡吧 阿门） 难受... 要么就弄个检测把 账号 用几个手机登录全部查出来也能实现（在打卡时把imie码也绑定上去 每个月查询经常换imie码的 ） 可是基本不会做这么傻逼的事 让手下人心不稳）
开始考虑如何避免不让同事帮忙打卡
换手机 手机丢失的情况不让后台删除数据 或者不需要再新创接口
这时候 手机序列号为唯一键 
登录的时候根据手机序列号 查询数据库 
if(数据== null){
	插入这个账号
}else{
	if(账号 == 账号){
	      跳入登录接口
	}else{
	      你想帮忙打卡？门都没
	}
}

扩展：目前用的打卡软件根据位置来打卡
应该是获取摸个地方的坐标  
把这个坐标写进打卡地点
然后你打卡的时候 获取你当前手机的定位判断位置坐标
这个只能说 手机定位 安卓苹果都能修改扎吧。
如果自己不懒写个脚本时间定时打开app打卡也不是不可能....



一、需求提出
   如今手机打卡被各公司应用,打卡软件的使用对人事部门统计绩效考勤很是方便,可是方便同时作弊也很多,同事帮忙打卡什么的各种出现,
 于是乎老板提出这个需求
二、基本思路
   作为一个开发者,程序逻辑有漏洞被利用就是自己的过失,简单的打卡软件就是登录,获取账号 打卡地点 时间, 还有打卡时间判断  判断你是上班打卡还是下班打卡,
 对于登录的限制 无非就是对手机限制只能登陆一个账户,关于ios与Android 获取 imie码来判断绑定
三：不完美点说明
   基本如果不是开发公司打卡,修改定位什么的这种骚操作 基本不会出现
